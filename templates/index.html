Flask API Documentation

This document provides detailed information about the available endpoints, request parameters, and responses for the User Authentication API.

Overview

This Flask server provides several endpoints for user authentication and validation. The API supports checking if a username or email exists, validating passwords, and this root endpoint for documentation.

CORS Configuration

The server implements Cross-Origin Resource Sharing (CORS) with the following configuration:

  • Origins: All origins are allowed (*)
  • Methods: GET, POST
  • Headers: Content-Type

User Data

The server has the following predefined users:

Username Email

Endpoints

GET /

Serves this documenation page.

Response

HTML

GET /user_exists

Checks if a username exists in the system.

Query Parameters

Parameter Type Required Description
username string Yes The username to check

Response

JSON object with the following properties:

{
  "username": "string",
  "exists": boolean
}

Example

Request: GET /user_exists?username=admin

{
  "username": "admin",
  "exists": true
}

Error Response

If the username parameter is missing:

{
  "error": "Missing 'username' parameter"
}

Status Code: 400 Bad Request

Note: This endpoint has an intentional delay of 0.5 seconds to simulate network latency.

GET /user_exists_unreliable

Similar to /user_exists but intentionally fails 50% of the time.

Query Parameters

Parameter Type Required Description
username string Yes The username to check

Response

When successful, returns the same response as /user_exists.

Error Response

50% of requests will result in a 500 Internal Server Error.

Note: This endpoint has an intentional delay of 0.5 seconds and is designed to be unreliable for testing error handling.

GET /email_registered

Checks if an email address is already registered in the system.

Query Parameters

Parameter Type Required Description
email string Yes The email address to check

Response

JSON object with the following properties:

{
  "email": "string",
  "exists": boolean
}

Example

Request: GET /email_registered?email=admin@someplace.com

{
  "email": "admin@someplace.com",
  "exists": true
}

Error Response

If the email parameter is missing:

{
  "error": "Missing 'email' parameter"
}

Status Code: 400 Bad Request

Note: This endpoint has an intentional delay of 0.5 seconds to simulate network latency.

POST /valid_password

Validates if a password meets the required criteria.

Password Requirements

  • Minimum length of 8 characters
  • At least one digit
  • At least one uppercase letter

Request Body

JSON object with the following structure:

{
  "password": "string"
}

Response

JSON object with the following properties:

{
  "valid": boolean,
  "message": "string"
}

Example

Request:

{
  "password": "Password123"
}
{
  "valid": true,
  "message": "Password is valid."
}

Note: This endpoint has an intentional delay of 0.5 seconds to simulate network latency.
